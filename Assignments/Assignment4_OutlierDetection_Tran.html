3. Output the outliers for the variable hum based on 2 standard deviation¶

As the distribution of the var hum is close to a normal distribution, then we can detect outliers using the standard deviation method.

Refer to Program 5.1 in Chapter 5

In [18]:
*Use PROC MEANS to Output means and standard deviations to a data set;
proc means data=mylib.bikeday noprint ;
    var hum;
   output out=Mean_Std(drop=_type_ _freq_)
          mean=
          std= / autoname;
run;

title "Outliers for hum Based on 2 Standard Deviations";
data _null_;
   file print;
   set mylib.bikeday (keep=instant hum );
   ***bring in the means and standard deviations;
   if _n_ = 1 then set Mean_Std;

   if hum lt hum_Mean - 2*hum_StdDev and not missing(hum) then put "low " instant= hum=;
   else if hum gt hum_Mean + 2*hum_StdDev then put "high " instant= hum=;
run;

4. Output the outliers for the variable hum based on the interquantile range method.¶

Refer to program 5.8 from chapter 5, print Q1, Q3 and the Interquantile range (IQR) and output the outliers for hum using the interquantile range method

In [19]:
title "Detect Outliers Based on Interquartile Range";
proc means data=mylib.bikeday noprint;
   var hum;
   output out=Tmp (drop=_type_ _freq_)
          Q1=
          Q3=
          QRange= / autoname;
run;
title "Quartiles and IQR for hum";
proc print data=Tmp; 
run;

title "Possible outliers";
data _null_;
   file print;
   set mylib.bikeday(keep=instant hum);
   if _n_ = 1 then set Tmp;
   if hum le hum_Q1 - 1.5*hum_QRange and not missing(hum) or
      hum ge hum_Q3 + 1.5*hum_QRange then
      put "Possible Outlier for instant " instant "Value of hum is " hum;
run;

8. Do you see any risk in removing the windspeed outliers found based on interquantile range method? justify your answer.¶

Things to consider the following in your reasoning:

  • Although they are called outliers, would you consider these as errors? A value of 0.5 for windspeed is equivalent to 0.5*67= 33km which is considerd as fresh breeze. https://en.wikipedia.org/wiki/Beaufort_scale
  • relate the risk of deleting those data points on the predictive model. The highest value of speedwind is 0.507463 and is associated with count=1635. This means as it becomes more windy, the count of rental bikes drops. some other outlier data points are also associated witn count<2000.
  • The scatter plot does not show any kind of linear relationship between windspeed and count. However if you decided to apply predictive analytics modeling on this dataset, it is better to remove the outlier.
